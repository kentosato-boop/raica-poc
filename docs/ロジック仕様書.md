## 1. システム構成（データの流れ）

```
Porters（正本DB）
   │ CSV日次同期（30,214候補者・求人・選考レコード）※API化は契約プラン確認中
   ▼
RAiCA DB（候補者・求人・企業・選考プロセス・接触ログ）
   │
   ├─ AI層
   │   ├ マッチング（類似度スコアリング：過去成約パターン照合）
   │   ├ 文面生成（3ペルソナ×3言語 / VI+日本語訳）
   │   ├ 予兆検知（成約履歴×外部ニュース×定期確認メール返信）
   │   └ 学習値（企業別平均返答日数・候補者別平均反応日数）
   │
   ├─ ルールエンジン（§4：3営業日/2営業日ルール → キュー生成）
   │
   └─ チャネル連携
       ├ Zalo OA（候補者向け個別配信・5,012フォロワー・Porters紐付62%）
       ├ Gmail（企業向け・推薦文）
       ├ Google カレンダー（面談確定→三者へ招待）
       └ Asana（タスク双方向同期・AI提案タスク自動起票）

すべての送信は HITL：AI下書き → 人間承認 → 送信 → ログをPorters選考レコードに紐付け
```

## 2. データモデル（主要エンティティ）

- **候補者** … Porters ID、状態（§3）、担当CA、希望条件（給与/通勤/勤務形態）、JLPT、並行応募（自社/他社別）、平均反応日数（AI学習値）、CA所感
- **求人** … 企業、受領日、レンジ給与、区分/業界、募集状態（募集中/急募/第2弾）、AI候補数（スコア70以上）
- **企業** … 過去成約履歴、採用傾向（面接回数・決定リードタイム）、NGパターン（夜勤辞退等）、平均返答日数（AI学習値）、再募集予兆シグナル
- **選考プロセス**（候補者×求人） … ステージ（§3）、イベント履歴（推薦日・面接日・提示条件）、催促/フォロー実施回数
- **接触ログ** … チャネル、トーン、送信者、承認者、既読/返信、→学習値の材料

## 3. 状態遷移（選考ステージ）

```
求人受領 → AI候補抽出 → 意向確認(CA) → 推薦(RA) → 書類選考 → 一次面接 → (二次) → 最終面接
                                                                        ├→ 企業OK ┐
                                                                        └→ 人材OK ┴→ 双方OK＝成約
どの段階でも：辞退/不成約 → 理由をNGパターンとして蓄積（次回マッチングの根拠に）
候補者状態：アクティブ／選考中（プロセスあり）／休眠（最終接触>1年）
```

**成約の定義＝「企業OK ∧ 人材OK」**。最終面接後は企業側と候補者側を並行トラッキングし、両方揃って初めて成約（だからRA/CA両画面が対になっている）。

## 4. ルールエンジン（キューの生成条件）

| | RA：書類選考催促キュー | CA：候補者フォローキュー |
|---|---|---|
| 起点 | 推薦日 | 候補者への接触日（面接実施・オファー提示等） |
| マスト連絡 | 3営業日 無返答で催促必須 | 2営業日 以内にフォロー必須 |
| 再送 | 3営業日おき・最大2回 | 2営業日おき・最大2回 |
| 上限後 | 電話に切替（トークメモ生成） | 電話でクロージング |
| AI上書き | 企業別「平均返答日数」超過で前倒し警告 | 候補者別「平均反応日数」超過で前倒し警告 |

**判定優先順位：AI学習値による前倒し ≧ 固定ルール**（例：Hoa様は平均反応1.5日→ルール上フォロー②は明日だが、AIは本日電話を推奨）。

## 5. マッチングロジック（推薦根拠の優先順位）

1. **成約類似**：この企業が過去に採用した人との経歴パターン類似度（%表示）
2. **NG照合**：この企業の不成約・辞退パターンに該当しないか
3. **原文引用**：職務経歴書・面談メモの該当箇所
4. スコア内訳（スキル/経験/日本語/給与/通勤）は**参考値** — 判断は1〜3で行う

- **要件ずらし**：機械適用なら落ちる候補を、前例（成約実績・本人申告）を根拠に緩和提案（例：N4→通訳常駐ライン限定、経験2年+N3→3年要件の緩和、通勤35分→本人許容45分）
- スコア70未満は非表示（求人ごとのAI候補数 − 表示数 ＝ 非表示数）

## 6. KPI定義と計算式（画面の全数値）

### 月次ファネル（ダッシュボード＝チーム全体・今月）
```
稼働企業84社 → 推薦128名 →(47.7%)→ 一次面談61 →(55.7%)→ 最終面談34
最終面談34 ─┬→ 企業OK26（76.5%）
            └→ 人材OK29（85.3%）  → 双方OK（成約）22（64.7%）
推薦→成約の展開率 = 22/128 = 17.2%
検算：26+29−22=33 ≤ 34（両側判断済み33名・両側待ち1名）
```

- 一次面談（実施）61 ＝ ファネルの一次面談

- **AI推薦の承認率 78.5% ＝ RA承認128 ÷ AI提示163**（AIの提示品質の指標）

### 個人目標（マイデスク・月次）
- ペースライン＝月経過率（7/17時点＝55%）。目標比が **+8pt以上→先行 / ±8pt→順調 / −8pt未満→遅れ**
